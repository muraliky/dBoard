UAT COE Dashboard

Quick reference to open and use the UAT COE Dashboard delivered as a static HTML bundle.

📦 Delivered files: index.html 📅 Updated: 2026-04-01
1 Before You Start
Platform
Windows 10 / 11 recommended
Browser
Microsoft Edge (Chromium)
Distribution
Files are shared via SharePoint. When users click the SharePoint link, SharePoint will download a local copy of the dashboard folder.
2 Download & Open

Click on the shared SharePoint link provided to you. An index.html file will be downloaded to your Downloads folder.

Locate the downloaded index.html file on your machine (usually in Downloads).

Double-click index.html OR right-click index.html → "Open with" → choose "Microsoft Edge".

If Edge is not the default browser, explicitly select Microsoft Edge for the first open.

If Windows offers to always use Edge for .html files, accept that to make future opens easier.

3 Login

After opening index.html, the dashboard will show the Login overlay (or you may be presented login.html).

Enter your KID (employee ID) into the Employee ID field. IDs are case-insensitive.

Click the Sign In / Login button.

If your KID exists in the registry, you will be signed in and navigated to the dashboard.

If your KID is not recognised, you will see an error: "Employee ID not recognised. Please contact your administrator." Contact the dashboard admin and provide your KID.

4 Access Levels
🔴 Superuser
Sees all groups and all data across the dashboard.
🟡 Group User
Sees data for their assigned group (e.g., Payments, WIM).
🔵 Subgroup User
Sees only the specific subGroup within a group.
5 Navigation

Use the left sidebar to switch between pages:

Quality Metrics
Automation Metrics
Automation Standardization
CATS (Initiatives)
Training
RTO (Return To Office)
People & Engagement

📄 Page Layout

Each page contains charts, tables, and filters. The header displays your user name and role badge in the top-right corner.

6 Filters & Controls

Available Filters

Group Sub Group Year Month Page-specific controls

Filters are locked or limited according to your role. For example, group users cannot change the group filter to view another team's data.

Use the chart legends and dropdowns to refine what you see. Use the refresh/load buttons (if present) to reload data.

7 Signing Out

Use the Logout button in the header to clear your session and return to the login overlay.

8 Troubleshooting
▶ Blank page or blocked scripts

⚠️ Cause: Browser may be blocking local file access for scripts.

✅ Fix: Open index.html in Microsoft Edge (Chromium). If you continue to face issues, clear your cookies and browsing history for the past 24 hours and reload the page.
▶ "Employee ID not recognised"

⚠️ Cause: KID not present in data/users.js registry.

✅ Fix: Confirm your KID and contact the dashboard administrator to add it.
▶ I prefer a browser other than Edge

ℹ️ Note: The dashboard is tested on Edge (Chromium). Other modern Chromium browsers (Chrome) may work, but Edge is recommended by default.
